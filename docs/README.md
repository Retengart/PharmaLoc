# PharmaLoc: Геомаркетинговый анализ для аптечного ритейла

Система автоматизированного поиска и оценки перспективных локаций для открытия аптек в мегаполисе. В основе лежит анализ пространственных данных OpenStreetMap (OSM) и ансамблевое машинное обучение.

## 🎯 Суть проекта
Выбор места для новой аптеки — это всегда риск. PharmaLoc снижает его, анализируя городскую среду: близость к метро, плотность жилой застройки, медицинскую синергию и активность конкурентов. Вместо субъективных экспертных оценок вы получаете **Integral Potential Score** для каждой точки города.

## 🛠 Технологический стек
*   **Сетка**: Uber H3 (разрешение 9, ~174м между центрами ячеек).
*   **Данные**: OpenStreetMap (через `OSMnx`), портал data.mos.ru.
*   **ML**: CatBoost и Random Forest с оптимизацией гиперпараметров через `Optuna`.
*   **Гео**: GeoPandas, Shapely, Folium (интерактивные карты).
*   **Производительность**: Распараллеливание вычислений через `pandarallel`.

## 🚀 Запуск за 5 минут

### 1. Установка зависимостей
Рекомендуется использовать виртуальное окружение Python 3.10+:
```bash
pip install -r app_pharmacy/requirements.txt
```

### 2. Запуск пайплайна
Скрипт автоматически скачает данные, обучит модель и сформирует отчеты:
```bash
python3 -m app_pharmacy.main --train --no-leakage
```
*Флаг `--no-leakage` исключает признаки, которые могут вызвать переобучение (например, дистанцию до самой себя).*

## 📊 Что на выходе?
После завершения работы в папке `data/` появятся:
1.  **potential_map.html** — интерактивная карта города с тепловым слоем потенциала и детальными подсказками.
2.  **top_10_locations_summary.csv** — список приоритетных адресов для полевого обследования.
3.  **Detailed Reports** — подробная статистика по каждой ячейке (трафик, жилье, конкуренты).

## 🧪 Методология расчета
Итоговый потенциал локации рассчитывается как гибрид двух оценок:
1.  **Rule-based**: экспертные веса факторов (медицина, транспорт, жилье).
2.  **ML-based**: вероятность успеха, предсказанная обученным классификатором.

Формула: `Score = 0.6 * Weights + 0.4 * ML_Probability`
