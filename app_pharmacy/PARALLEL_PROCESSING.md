# Параллельная обработка данных в проекте

## Обзор

Проект использует параллельную обработку для ускорения вычислений. Python поддерживает параллелизм через несколько механизмов:

1. **concurrent.futures** - высокоуровневый API для параллелизма
2. **multiprocessing** - для CPU-bound задач
3. **threading** - для I/O-bound задач
4. **numpy vectorization** - векторизованные операции

## Реализованные оптимизации

### 1. Параллельная загрузка данных OSM (`parallel_load_osm_data`)

**Тип**: ThreadPoolExecutor (I/O-bound операции)

**Что делает**: Загружает разные типы объектов OSM параллельно (аптеки, транспорт, жилые здания и т.д.)

**Ускорение**: ~3-5x при загрузке 8 типов объектов

```python
osm_data = parallel_processing.parallel_load_osm_data(roi_geometry)
```

### 2. Векторизованный расчет признаков (`vectorized_distance_features`)

**Тип**: NumPy vectorization

**Что делает**: Использует векторизованные операции NumPy вместо циклов Python для расчета расстояний

**Ускорение**: ~10-50x для больших сеток H3 (зависит от количества ячеек и POI)

**Алгоритм**: Формула гаверсинуса в векторизованном виде

```python
h3_grid = parallel_processing.vectorized_distance_features(
    h3_grid, poi_data, 'transport'
)
```

### 3. Параллельная обработка ячеек (`parallel_calculate_distance_features`)

**Тип**: ProcessPoolExecutor (CPU-bound операции)

**Что делает**: Обрабатывает каждую ячейку H3 в отдельном процессе

**Ускорение**: ~N_JOBS раз (где N_JOBS = количество ядер CPU - 1)

**Примечание**: Используется как альтернатива векторизованному методу для очень больших данных

### 4. Параллельный расчет нескольких признаков (`parallel_calculate_multiple_features`)

**Тип**: ThreadPoolExecutor

**Что делает**: Рассчитывает несколько типов признаков одновременно

**Ускорение**: Зависит от количества типов признаков

## Производительность

### Без параллелизации:
- Загрузка OSM данных: ~30-60 секунд (последовательно)
- Расчет признаков для 154 ячеек: ~5-10 минут

### С параллелизацией:
- Загрузка OSM данных: ~10-15 секунд (параллельно)
- Расчет признаков для 154 ячеек: ~30-60 секунд (векторизовано)

**Общее ускорение**: ~5-10x для типичного случая

## Настройка

Количество используемых ядер CPU определяется автоматически:

```python
N_JOBS = max(1, mp.cpu_count() - 1)  # Оставляем одно ядро свободным
```

Можно изменить в `app_pharmacy/parallel_processing.py`:

```python
N_JOBS = 4  # Фиксированное количество ядер
```

## Ограничения

1. **GIL (Global Interpreter Lock)**: 
   - ThreadPoolExecutor не ускоряет CPU-bound задачи из-за GIL
   - Используется только для I/O операций (загрузка данных)
   - ProcessPoolExecutor обходит GIL

2. **Память**:
   - Параллельная обработка требует больше памяти
   - Для очень больших данных может потребоваться chunking

3. **Overhead**:
   - Для маленьких задач overhead может превысить выигрыш
   - Векторизация эффективна даже для небольших данных

## Рекомендации

1. **Для маленьких ROI** (< 100 ячеек): Используйте векторизованные функции
2. **Для средних ROI** (100-1000 ячеек): Используйте векторизацию + параллельную загрузку
3. **Для больших ROI** (> 1000 ячеек): Используйте все методы параллелизации + chunking
